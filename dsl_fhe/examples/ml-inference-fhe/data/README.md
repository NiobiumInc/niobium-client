# ml-inference weights

This example runs an encrypted 2-layer MLP (784→512→10) on MNIST digits. It
needs two things that are **not vendored** in this open-source client repo:

1. **The HEIR-generated model** — a ~28K-line OpenFHE forward pass
   (`mlp_openfhe.cpp`, generated by Google's HEIR compiler). A no-op **stub**
   ships here (`../mlp_openfhe.{h,cpp}`) so the example builds and runs
   self-contained; it does **not** compute real inference.

2. **Trained weights** — `fc1_weight.bin`, `fc1_bias.bin`, `fc2_weight.bin`,
   `fc2_bias.bin` (little-endian float32). These are likewise not vendored.

## Build & run with the stub (default, self-contained)

`make ml-inference` builds against the stub model and generates **stub weights**
(zeros) into this directory via `make_stub_weights.py`. The pipeline links and
runs end-to-end but the output is not meaningful inference. You can also
generate the stub weights manually:

```bash
python3 examples/ml-inference-fhe/data/make_stub_weights.py
```

## Run real inference (opt-in)

Obtain the real HEIR-generated sources and trained weights from the
ml-inference submission package (the open-source ML inference repo / submission
that ships `mlp_openfhe.cpp` + `data/*.bin`), then build against them:

```bash
make ml-inference SUBMISSION_REPO=/path/to/submission-repo
```

When `SUBMISSION_DIR` resolves to a real submission, its `src/mlp_openfhe.cpp`
and `data/` take precedence over the local stub (see the Makefile and
`examples/ml-inference-fhe/HOWTO.md`).
